# har-sanitizer

A standalone, single-file build of [Cloudflare's HAR Sanitizer](https://github.com/cloudflare/har-sanitizer) — a tool for redacting sensitive information (cookies, authorization headers, tokens/JWTs, query and POST parameters, and embedded content) from HAR (HTTP Archive) files before sharing them for debugging or support. This is not a custom tool — it's a packaging of the existing Cloudflare software into one portable HTML file.

Google's original `har-sanitizer` is archived and depends on Python 2.7, so the actively maintained Cloudflare fork is used instead. All sanitization happens in the browser, so the whole tool collapses cleanly into a single file with no server.

## Features

- Redacts cookies, headers, URL/POST/form parameters, and content mimetypes
- Choose exactly which values to scrub before exporting
- Runs **entirely client-side** — your HAR file never leaves your machine
- Works fully offline, opened straight from the filesystem (no install, no server)

## Usage

Open `har_sanitizer.html` directly in a browser. Drag in (or select) a `.har` file, choose what to redact, and download the sanitized result.

## Building / Updating

The HTML is built from upstream in an ephemeral Docker container, so nothing is installed on the host. You only need Docker.

```bash
./build.sh            # build from the latest upstream commit (main)
./build.sh <ref>      # build from a specific commit SHA, tag, or branch
```

`build.sh` runs a Node container that clones `cloudflare/har-sanitizer`, builds it with [`vite-plugin-singlefile`](https://github.com/richselian/vite-plugin-singlefile) (which inlines all JS/CSS so the page works over `file://`), and writes `har_sanitizer.html` back into this folder. A provenance comment recording the exact source commit and build date is prepended to the output.

After rebuilding, sanity-check the result by opening `har_sanitizer.html` and loading `test/sample.har` — it contains known dummy secrets (cookies, an `Authorization` header, an `api_key` query param, and a token in the response body). With all categories selected, every one of those values should come back `[... redacted]`.

## Files

- `har_sanitizer.html` — the tool (generated by `build.sh`).
- `build.sh` — container-based build script.
- `vite.singlefile.config.ts` — Vite config override that adds the single-file plugin to upstream's build.
- `test/sample.har` — a small HAR with dummy secrets for verifying a build.
- `README.md` — this file.
